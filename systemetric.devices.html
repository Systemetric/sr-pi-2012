devices Package¶

All the non-sr devices that are or can be plugged into the robot

Mbed Class¶

The mbed itself, and basic communications with it

classmethod get()¶

A lazy singleton, using the default port. This should be used instead of calling the constructor, to prevent the serial port being opened twice

sendCommand(c)¶

Send a short command, and return a single line response. Prevents other threads interweaving requests by locking on self._lock

MbedDevice Class¶

class systemetric.devices.MbedDevice(id, mbed=None)¶

A device attached to an mbed. The protocol is to send a single character representing the device ID, then another (if necessary) to specify what information should be retrieved. Arguments can be given in a function- like syntax. For example::

MbedDevice('G').request('r', 10) -> Gr(10)
MbedDevice('C').request('h')      -> Ch

The mbed then returns a single line response. The line may be empty

Request something from the mbed

Arm Class¶

class systemetric.devices.Arm(mbed=None)¶

Bases: systemetric.devices.mbed.MbedDevice

A class to control the arm via the mbed. The arm has two control wires - a blue one, which causes the arm to move when pulled to 0V (through a FET), and a yellow one, which is pulled to 0V when the bottom limit switches are pressed.

Determines if either bottom limit switch is pressed

atTop¶

Grabs a cube, optionally waiting for the grabber to return to its starting position.

Compass Class¶

class systemetric.devices.Compass(mbed=None)¶

Bases: systemetric.devices.mbed.MbedDevice

A class to communicate with the compass via the mbed

absoluteHeading¶

Get the compass heading from the mbed, measured from due north. Returns NaN upon error

Type :Bearing
heading¶

Get the compass heading from the mbed, relative to the offset. Returns NaN upon error.

When set, adjusts the zero calibration of the compass such that an immediate read of heading gives the same value as it was set to.

Type :Bearing
startCalibration()¶

Start calibrating the compass. While this is happenning, the compass should be spun around slowly for two revolutions, taking about 20 seconds

stopCalibration()¶

Exit calibration mode

Bases: systemetric.devices.mbed.MbedDevice

A class to communicate with the gyro via the mbed

angularVelocity¶

Get the angular velocity measured by the gyro. Approximately calibrated to degrees per second

calibrate()¶

Calibrate the zero offset of the compass, by sampling and averaging a lot of readings. This should be called frequently, as the voltage supplied to the mbed appears to change, changing the response of the sensor.

LaserGate Class¶

class systemetric.devices.LaserGate(mbed=None)¶

Bases: systemetric.devices.mbed.MbedDevice

A class to read the laser light gate via the mbed. The laser needs to be powered with 3.3V from the mbed, and some kind of light detector should be hooked up as well. Might be a good idea to only turn the laser on when the sensor is checked.

Determine if the light gate is obstructed
